## Internet-Verbindung

### Bandbreiten-Anforderungen

| Nutzung                 | Minimum | Empfohlen | Hinweis                   |
| ----------------------- | ------- | --------- | ------------------------- |
| **Allgemein**           |         |           |                           |
| Initial Load            | 2 Mbps  | 5 Mbps    | ~500KB Assets (gzipped)   |
| Normale Nutzung         | 2 Mbps  | 5 Mbps    | JSON API calls (~1-10 KB) |
| **Bild-Upload**         |         |           |                           |
| Einzelnes Bild          | 3 Mbps  | 10 Mbps   | Max 2MB pro Bild          |
| Mehrere Bilder parallel | 5 Mbps  | 10+ Mbps  | Parallele Uploads         |
| **Latenz**              |         |           |                           |
| Optimal                 | <50ms   | -         | Server in Europa          |
| Akzeptabel              | <200ms  | -         | Normale Verbindung        |
| Grenze                  | <500ms  | -         | Spürbare Verzögerungen    |

### Offline-Modus

- ❌ **Nicht unterstützt**
- Alle Features erfordern aktive Internet-Verbindung:
  - Login/Registrierung
  - Termin-Buchung
  - Praxis-Suche
  - Profil-Bearbeitung
  - Bild-Upload

**Zukünftige Features (in Entwicklung):**

- Native Mobile App (iOS & Android)
- Offline-Modus in Mobile App (Termin-Cache, Profil-Ansicht)

## Zusätzliche Anforderungen

### JavaScript

- **Status:** ✅ Zwingend erforderlich
- **Grund:** React-App (Single Page Application)
- **Fallback:** Fehlerseite "JavaScript erforderlich" wird angezeigt
- **Standard:** JavaScript ist bei allen modernen Browsern standardmäßig aktiviert

### Cookies

- **Status:** ✅ Zwingend erforderlich für Login
- **Cookie:** `access_token` (HttpOnly, Secure)
- **Gültigkeitsdauer:** 10 Minuten (wird automatisch erneuert)
- **Fallback:** Login nicht möglich, Fehler "Cookies müssen aktiviert sein"

### localStorage (optional)

- **Status:** ⚠️ Optional (für bessere Nutzererfahrung)
- **Verwendung:**
  - Suchfilter merken (z.B. Tierart, Service)
  - UI-Präferenzen
- **Fallback:** App funktioniert, aber Filter/Präferenzen gehen bei Seitenneuladen verloren

### Popup-Blocker

- **Status:** ⚠️ Kann Features blockieren
- **Betroffene Features:** Praxis-Websites (öffnen in neuem Tab)
- **Lösung:** Popup-Blocker für vetilib.de erlauben

## Nicht-unterstützte Browser

### Komplett nicht unterstützt

| Browser/Gerät            | Grund                               | Alternative                        |
| ------------------------ | ----------------------------------- | ---------------------------------- |
| Internet Explorer (alle) | Veraltet, Microsoft Support beendet | Chrome, Edge, Firefox installieren |
| Opera Mini               | JavaScript-Einschränkungen          | Opera Mobile oder Chrome verwenden |
| Sehr alte Browser        | Chrome <90, Firefox <88, Safari <14 | Browser aktualisieren              |

### Eingeschränkt unterstützt

| Browser/Gerät                 | Problem                    | Lösung                                      |
| ----------------------------- | -------------------------- | ------------------------------------------- |
| Safari 14-16                  | Teilweise CSS-Bugs         | Auf Safari 17+ aktualisieren                |
| Android 7-9                   | Langsame Performance       | Auf Android 10+ upgraden oder neueres Gerät |
| Sehr kleine Displays (<375px) | UI teilweise abgeschnitten | Größeres Gerät verwenden                    |

### Hinweise

- **JavaScript deaktiviert:** App lädt nicht, zeigt Fehlerseite
- **Cookies blockiert:** Login nicht möglich, zeigt Fehler
- **Alte Browser:** Features können fehlen, Layout-Probleme, Sicherheitsrisiken

## Quick Check

### ✅ VetLib System-Check

**Vor dem ersten Start prüfen:**

**Browser:**

- [ ] Chrome 90+ / Safari 14+ / Firefox 88+ / Edge 90+ installiert?
- [ ] JavaScript aktiviert? (Standard: aktiviert)
- [ ] Cookies erlaubt? (Standard: erlaubt)

**Gerät:**

- [ ] Bildschirmgröße: Mind. 375px Breite (Smartphone) oder 1200px (Desktop)?
- [ ] Modernes Gerät? (nicht älter als ~5 Jahre)

**Internet:**

- [ ] Aktive Internet-Verbindung?
- [ ] Bandbreite: Mind. 2 Mbps? (Test: fast.com oder speedtest.net)

**Optional:**

- [ ] localStorage aktiviert? (für Filter-Speicherung)
- [ ] Popup-Blocker für vetilib.de erlaubt? (für Praxis-Websites)

### Schnelltest

Öffne https://vetilib.de → Wenn Startseite lädt: ✅ Alles OK!

## Troubleshooting

### Häufige Probleme

| Problem                               | Mögliche Ursache             | Lösung                                                  |
| ------------------------------------- | ---------------------------- | ------------------------------------------------------- |
| Seite lädt nicht                      | Internet-Verbindung fehlt    | WLAN/Mobile-Daten prüfen                                |
| Seite zeigt "JavaScript erforderlich" | JavaScript deaktiviert       | Browser-Einstellungen → JavaScript aktivieren           |
| Login funktioniert nicht              | Cookies blockiert            | Browser-Einstellungen → Cookies für vetilib.de erlauben |
| Layout sieht komisch aus              | Sehr alter Browser           | Browser aktualisieren (Chrome, Firefox, Safari)         |
| Sehr langsam                          | Langsame Internet-Verbindung | Internet-Geschwindigkeit testen (speedtest.net)         |
| Bilder laden nicht hoch               | Upload zu groß (>2MB)        | Bild komprimieren oder kleineres Bild wählen            |
| Fehler "Access Denied"                | Session abgelaufen           | Neu einloggen                                           |
| Praxis-Website öffnet nicht           | Popup-Blocker                | Popup-Blocker für vetilib.de deaktivieren               |

### Browser aktualisieren

- **Chrome:** https://www.google.com/chrome/
- **Firefox:** https://www.mozilla.org/firefox/
- **Safari:** macOS System-Update (Systemeinstellungen → Softwareupdate)
- **Edge:** Windows Update oder https://www.microsoft.com/edge

### JavaScript aktivieren

- **Chrome:** Einstellungen → Datenschutz und Sicherheit → Website-Einstellungen → JavaScript
- **Safari:** Einstellungen → Websites → JavaScript
- **Firefox:** about:config → javascript.enabled → true

### Cookies aktivieren

- **Chrome:** Einstellungen → Datenschutz und Sicherheit → Cookies → "Cookies von Drittanbietern blockieren" AUS
- **Safari:** Einstellungen → Datenschutz → "Alle Cookies blockieren" AUS
- **Firefox:** Einstellungen → Datenschutz → "Cookies und Website-Daten" → Erlauben
