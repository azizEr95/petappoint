## Seed-Skripte Details

### 1. seed-static.ts

**Typ:** Seed (Persistente Basisdaten)

**Ausführung:**
- `npm run seed:static` (aus backend root)
- `npm --prefix ./backend run seed:static` (aus root)

**Zeitpunkt:** Einmalig nach fresh DB setup oder manueller reset

**Was wird erstellt:**
- **15 Veterinary Practices** (Berlin, Hamburg, München, Köln, Frankfurt, Stuttgart, Düsseldorf, Leipzig, Dresden, Hannover, Nürnberg, Bremen, Essen, Dortmund, Bonn)
- **~40 Veterinarians** (2-3 je Praxis, randomisierte Namen)
  - Je vet: 3-5 random services + 2-4 random animal types
- **6 Test-Accounts** (joe@doe.de, daniel@daniel.de, james@jay.de, m@m.de, joey@doey.de, aziz@erol.de)
- **24 Animals** (1 pro Tierrasse, verteilt auf 7 Tiertypen)
- **Foundation Data:** Services (11), Animal Types (7), Animal Races (24), Animal Groups (5)

**Phasen:**
1. **Foundation Phase:** Services, Animal Types, Races, Groups (idempotent)
2. **Practices Phase:** 15 Praxen mit addresses (email-check)
3. **Test Persons Phase:** 6 fixed test accounts (email-check)
4. **Veterinarians Phase:** ~40 vets verteilt auf praxen (email-check)
5. **Animals Phase:** 24 animals mit person-animal links (name+owner-check)
6. **Specialization Phase:** Vets zu services + animal types linken

**Idempotenz:**
- Alle Entities werden via email/eindeutiger ID gecheckt
- Existierende Daten werden übersprungen
- Confirmation codes + verification aktiviert
- **Passwort:** `Vetilib123!` (auto-hashed via Prisma extension)

**Besonderheiten:**
- Veterinarians als Person + zusätzliche Veterinarian-Relation
- Confirmation codes: `222222` (verified)
- Addresses mit genauen lat/lng Koordinaten
- Animals: sex=female, lifestyle=indoor, dateOfBirth=2021-01-13, weight=4000g, isCastrated=true

### 2. seed-dynamic.ts

**Typ:** Seed (Datumsrelative Termine)

**Ausführung:**
- `npm run seed:dynamic` (aus backend root)
- `npm --prefix ./backend run seed:dynamic` (aus root)

**Zeitpunkt:** Periodisch zum Aktualisieren der Termine oder nach static seed

**Was wird erstellt:**
- **~4500 Appointments** (destruktiv: löscht alte appointments/service links)
- **Past appointments (-14 bis -1 Tage):** ~1000 total, 35% mit animal linkage
- **Future appointments (+1 bis +28 Tage):** ~3500 total, 25% booked, 75% available
- **Appointment-Service Links:** basierend auf vet specializations

**Logik:**

1. **Fetch:** Alle practices, vets, services, animals aus DB
2. **Delete:** Alte appointments + service links (transaction)
3. **Past Appointments:**
   - Loop -14 bis -1 Tage
   - Weekday (Mo-Fr): 5-8 appointments/tag
   - Weekend (Sa-So): 2-4 appointments/tag
   - Uhrzeiten: 8:00-17:59
   - Minuten: 0 oder 30
   - Duration: 30-75 Minuten (30 + 0-2 × 15min)
   - 35% mit animal linkage
4. **Future Appointments:**
   - Loop +1 bis +28 Tage (Praxen 11-15: starten bei +15)
   - Weekday (Mo-Fr): 8-12 appointments/tag
   - Weekend (Sa-So): 4-6 appointments/tag
   - Uhrzeiten: 8:00-17:59, Minuten: 0 oder 30
   - 25% booked (mit animal), 75% available (frei)
5. **Batch Create:** Appointments in 1000er-batches
6. **Service Links:**
   - Available appointments: ALL vet services verlinkt
   - Booked appointments: ONE random vet service verlinkt

**⚠️ Destruktiv:** Löscht alle existing appointments! Nicht für production data!

## Ausführungsflows

### Flow 1: Initial Project Setup

```bash
npm install              # Root dependencies
npm run installProject   # All workspaces
npm run build           # Build shared
```

### Flow 2: Development Startup (Complete Stack)

```bash
# ONE COMMAND:
npm run vetilib

# ODER manuell:
npm run vetilib:detached
npm run migrateDB
npm run seed
npm run startBackendFrontend
```

**Result:** DB + Backend (3000) + Frontend (3001)

### Flow 3: Fresh Database

```bash
npm run stop
npm run vetilib:detached
npm run migrateDB
npm run seed
```

### Flow 4: Update Only Appointments

```bash
npm run seed:dynamic    # Delete + recreate appointments
```

### Flow 5: Backend Testing

```bash
cd backend
npm test                # Integration tests
# ODER
npm run test:ui         # Mit UI
```

### Flow 6: E2E Testing

```bash
# Auto-seeded via globalSetup:
npm run e2e-tests:ci    # Playwright tests (recommended)
```

## Troubleshooting

### "npm run seed:static" fails: "Germany not found"

**Ursache:** init.sql nicht executed (DB hat keine countries)

**Lösung:**
```bash
npm run stop
npm run vetilib:detached
npm run migrateDB
npm run seed
```

### Docker port 5432 bereits in use

**Lösung:**
```bash
docker ps -a
docker stop <container-id>
docker rm <container-id>
npm run vetilib:detached
```

### Appointments nur -14 bis +28

**Design:** Historische Daten sind begrenzt (für history-view realistic)

### E2E tests fail: "Data mismatch"

**Ursache:** seed-e2e-tests.ts data wurde verändert

**Lösung:**
```bash
npm run seed:e2e
npm run e2e-tests:ci
```
